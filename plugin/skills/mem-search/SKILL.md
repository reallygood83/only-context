# 기억 검색 스킬 (Memory Search Skill)

지식 베이스(당신의 코딩 역사책)를 탐색하여 현재 문제 해결에 필요한 실마리를 찾습니다.

## 언제 사용하나요? (When to Use)

- **과거의 해결책이 필요할 때**: "아, 이거 저번에도 떴던 에러인데... 어떻게 고쳤더라?"
- **결정의 이유가 궁금할 때**: "왜 우리가 이 라이브러리를 쓰기로 했지?"
- **좋은 코드를 재사용하고 싶을 때**: "그때 짰던 기가 막힌 정규식이 어디 있더라?"
- **파일의 역사를 알고 싶을 때**: "이 파일은 원래 무슨 용도였지?"
- **최근 작업 내용이 기억 안 날 때**: "내가 어제 어디까지 했더라?"

## 사용법 (Usage)

찾고 싶은 내용을 자연스럽게 물어보세요. 예시는 다음과 같습니다:

```
/mem-search 인증 관련 에러 고친 기록 찾아줘
/mem-search 데이터베이스 스키마 결정 사항 검색해줘
/mem-search 캐싱 처리는 어떻게 했었지?
/mem-search 이 프로젝트의 최근 세션 내용 보여줘
```

## 작업 흐름 (Workflow)

1. **질문 분석하기 (Analyze the Query)**
   - 사용자가 무엇을 찾고 있는지 탐정처럼 단서를 잡습니다. (에러? 결정? 단순 정보?)
   - 어떤 프로젝트에 대한 것인지 파악합니다.

2. **검색 전략 세우기 (Search Strategy)**
   - `mem_search` 도구를 적절한 필터와 함께 사용합니다.
   - 에러를 찾는다면: `type: error` 필터 장착!
   - 결정을 찾는다면: `type: decision` 필터 장착!
   - 패턴을 찾는다면: `type: pattern` 필터 장착!

3. **결과 보여주기 (Present Results)**
   - 찾은 보물들을 알기 쉽게 요약해서 보여줍니다.
   - 핵심 내용(Snippet)을 살짝 보여주어 맞는지 확인시킵니다.
   - "전체 내용을 읽어드릴까요?" (`mem_read`) 하고 친절하게 물어봅니다.

## 출력 형식 (Output Format)

결과는 깔끔하고 보기 좋게 보여줍니다:

```markdown
## "검색어"에 대한 검색 결과 X건을 찾았습니다

### [제목] (유형, 날짜)
> 노트에서 발췌한 핵심 내용이 여기에 표시됩니다...
**프로젝트**: project-name | **태그**: #tag1 #tag2

---

이 중 자세히 보고 싶은 내용이 있으신가요? 말씀만 하세요!
```

## 고급 사용법 (Advanced Usage)

프로젝트 전체의 맥락을 알고 싶다면:
```
mem_project_context({ project: "project-name" })
```
이걸 쓰면 최근 세션, 아직 못 고친 에러, 현재 유효한 결정 사항들을 한 방에 브리핑해줍니다.
