# 기억 저장 스킬 (Memory Save Skill)

Claude Code의 기억 시스템(당신의 전용 코딩 일기장)에 지식을 영구적으로 저장합니다.

## 언제 사용하나요? (When to Use)

- **중요한 결정을 내렸을 때**: "왜 이 라이브러리를 선택했지?" 나중에 후회하지 않도록 기록합니다.
- **유용한 패턴을 발견했을 때**: "오, 이 코드는 나중에 또 써먹어야겠어!" 싶을 때 저장합니다.
- **지독한 오류를 해결했을 때**: 다시는 같은 실수를 반복하지 않기 위해 해결책을 적어둡니다.
- **새로운 것을 배웠을 때**: 오늘의 깨달음을 미래의 나에게 전송합니다.
- **나만의 노하우를 쌓을 때**: 프로젝트 전반에 걸친 지식을 정리합니다.

## 사용법 (Usage)

저장할 내용과 함께 명령어를 입력하세요. 예시는 다음과 같습니다:

```
/mem-save decision: 이 프로젝트에는 MongoDB 대신 PostgreSQL을 쓰기로 결정했어. (이유 포함)
/mem-save pattern: 날짜 파싱을 위한 완벽한 정규식 패턴: /\d{4}-\d{2}-\d{2}/
/mem-save learning: 이 API는 분당 100회 요청 제한이 있다는 걸 알게 됨.
/mem-save error-fix: CORS 문제는 헤더에 origin을 추가해서 해결했음.
```

## 노트 유형 (Note Types)

| 유형 (Type) | 용도 (Use For) | 비유 (Metaphor) |
|---|---|---|
| `decision` | 아키텍처나 설계 결정 및 그 근거 | 건물의 설계도 |
| `pattern` | 재사용 가능한 코드, 정규식, 알고리즘 | 요리 레시피 |
| `error` | 오류 해결 방법 및 우회로 | 오답 노트 |
| `learning` | 일반적인 통찰 및 지식 | 수업 필기 |
| `file` | 특정 파일에 대한 설명 및 문서화 | 사용 설명서 |

## 작업 흐름 (Workflow)

1. **내용 파악하기 (Identify Content Type)**
   - 사용자의 말 속에서 어떤 종류의 지식인지 파악합니다. (결정? 패턴? 깨달음?)
   - 무엇인지 헷갈린다면, 사용자에게 정중히 물어봅니다.

2. **구조 잡기 (Structure the Content)**
   - 핵심 내용을 추려냅니다.
   - 문맥(프로젝트, 태그)을 덧붙여 찾기 쉽게 만듭니다.
   - Obsidian이 좋아하는 Markdown 형식으로 예쁘게 포장합니다.

3. **저장하고 확인받기 (Save and Confirm)**
   - `mem_write` 도구를 사용해 실제로 저장합니다.
   - 저장이 잘 되었다면, 파일 경로를 보여주며 안심시켜줍니다.
   - "더 자세히 적어둘까요?"라고 물어보는 센스를 발휘합니다.

## 내용 템플릿 (Content Templates)

### 결정 (Decision)
```markdown
## 맥락 (Context)
[왜 이런 결정이 필요했나요?]

## 결정 사항 (Decision)
[무엇을 결정했나요?]

## 근거 (Rationale)
[왜 이 방법을 선택했나요? 다른 대안은 없었나요?]

## 결과 (Consequences)
- 긍정적 측면: ...
- 부정적 측면: ...
```

### 패턴 (Pattern)
```markdown
## 설명 (Description)
[이 패턴은 무엇을 하나요?]

## 사용 시기 (When to Use)
[어떤 상황에서 쓰면 좋은가요?]

## 구현 (Implementation)
\`\`\`language
[코드 예시]
\`\`\`
```

### 오류 해결 (Error Fix)
```markdown
## 문제 (Problem)
[발생한 오류 내용]

## 원인 (Cause)
[근본적인 원인은 무엇인가요?]

## 해결책 (Solution)
[어떻게 고쳤나요?]

## 예방 (Prevention)
[다시는 안 생기게 하려면?]
```

## 가이드라인 (Guidelines)

- **이유(Why)를 꼭 적어주세요**: 나중에 보면 '왜 그랬더라?' 하는 경우가 많습니다.
- **태그를 달아주세요**: 도서관의 책 분류표처럼, 나중에 찾기 쉽게 만들어줍니다.
- **관련된 노트가 있다면 연결해주세요**: 지식의 그물망을 촘촘하게 만듭니다.
- **나중에도 알아볼 수 있게 구체적으로 적어주세요**: 미래의 나를 배려합시다.
